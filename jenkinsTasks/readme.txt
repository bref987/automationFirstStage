Клонировать проект https://github.com/vitalliuss/helloci
Запускать тесты из проекта в директори Java с помощью цели mvn test

(запускать периодически - */5 * * * *)

Bring it on
1. Установите Jenkins

2. Создайте задачу которая будет делать следующее:

Клонировать проект https://github.com/vitalliuss/helloci
Запускать тесты из проекта в директори Java с помощью цели mvn test
(job - настройки - сборка - вызвать цели Maven верхнего уровня - цели test)
3. Настройте билд триггеры:

Запуск тестов не позднее чем через 5 минут после коммита в git 
(опрашивать SCM об изменениях - H/5 * * * *)
Каждый будний день в полночь (запускать периодически - 0 0 * * 1-5)
4. Опубликуйте файл “Java\target\surefire-reports\com.github.vitalliuss.helloci.AppTest.txt” как артефакт
(послесборочные артефакты - заархивировать артефакты - указать файлы для архивации) 

Данное задание выполняется на основе Bring It On задачи.

Hurt me plenty
1. Сменить порт сервера на 8081
(отключить службу - сменить номер порта в jenkins.xml - сохранить - перезапустить службу)

2. Создать ноду и настроить сервер так, чтобы джоба выполнялась только на slave ноде 
(создать новую ноду: новый узле - название узла - permanent agent - указать корень удаленно фс -
указать метки - способ запуска (launch agent by connecnting it to the master))
(настройки job - ограничить лейблы сборщиков, которые могут исполнять данную задачу - название нового узла)

3. Настроить Job Config History и thinBackup (скачать плагины)

Hardcore
Данное задание выполняется на основе Hurt Me Plenty

1. Создать пользователя user и дать ему права на просмотр джоб Jenkins, 
но без возможности записи или смены настроек

2. Создать параметризованную джобу HelloUser, 
которая будет спрашивать в качестве параметра имя пользователя (username) и писать в консоль "Hello, username!".

3. С помощью цели  mvn cobertura:cobertura измерьте покрытие кода юнит-тестами 
(code coverage) и опубликуйте на странице джобы в виде графика
